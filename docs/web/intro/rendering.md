# 浏览器渲染

目前主流框架 vue 还是 react 都是 spa 单页面应用，指的是整个网站页面只有一页 html 应用，使用路由来进行页面切换更新。  

浏览器渲染类型分为 
- `spa (Single Page Application)` 单页面渲染.
- `mpa (MultiPage Application)` 多页面渲染。

根据渲染的执行者可以分为 
- 客户端渲染 `csr (Client Side Render)` 
- 服务端渲染 `ssr (Server Side Render)` 

## 单页面渲染 spa

客户端拿到所有资源，请求 http 接口资源后进行渲染。
- 优点
  - 页面渲染在客户端执行，对服务端压力较小。
  - 页面不会有刷新，不会产生刷新时页面切换的白屏。
  - 一次加载全部公共资源，切换页面不需要再次请求。
  - 由于单页面，页面之间数据传递方便。
  - 对于开发者页面之间数据传值方便。
- 缺点
  - 首次访问会便会加载全部的代码，造成首次加载缓慢。
  - 并且加载完页面后，会调用页面的 http 服务去后端获取数据。造成两次请求。
  - 搜索引擎优化（seo）不好做。


:::tip

为了解决单页面渲染的 seo 以及首次加载缓慢的问题，所以就出现了预渲染以及服务端渲染这两种解决方式

:::


## 预渲染 prerendering
在构建过程中生成 html 的静态页面，客户端进行访问时，可以将渲染好的静态 html 传给客户。构建时便生成多个 html 静态页面，可以对单个页面进行搜索引擎优化。

- 优点
  - 构建时提前生成静态 html，不需要到客户端渲染生成，提高了首屏加载速度。
  - 搜索引擎可以直接解析 html 页面，更好的搜索引擎优化。
  - 快速的页面响应与路由。
- 缺点
  - 预渲染需要存储大量静态 html 页面，会占用一定存储空间。
  - 对于需要动态或频繁更新的页面不合适。
  - 会增加构建时间。

## 服务端渲染 ssr
在服务端来渲染生成 html 页面，客户端进行访问时，服务端会预先处理页面中的请求与数据，编译为 html 页面，将页面发送给客户端

- 优点
  - 可以对所有页面进行搜索引擎优化。
  - 对客户端的负载很小。
- 缺点
  - 动态的生成返回页面会使服务器负载增加。
